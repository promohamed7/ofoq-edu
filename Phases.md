# Development Principles

These rules are mandatory.

## DO

- Build incrementally.
- Keep commits small and meaningful.
- Verify every completed feature using TestSprite.
- Fix the root cause, not the symptom.
- Update LOOP.md after every successful verification cycle.
- Keep README updated when needed.

## DO NOT

- Rewrite the entire project.
- Migrate to another framework.
- Implement multiple unrelated features at once.
- Skip verification.
- Skip documentation.

# Decision Log

Before implementing any feature, evaluate:

1. Does it improve Project Quality?
2. Does it improve Loop Quality?
3. Does it improve Innovation?
4. Does it justify a new TestSprite verification loop?

If the answer is "No" to all, do not implement it.
